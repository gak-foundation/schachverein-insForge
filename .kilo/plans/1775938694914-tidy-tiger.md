## Kritische Lücken vor öffentlichem Launch

### 1. TRF-Import/Export (Release 2 - Must)
**Warum kritisch:** SwissChess ist Standard-Software für Schachturniere. Ohne TRF-Import können keine existierenden Turnierdaten migriert werden.

**Technische Umsetzung:**
- TRF-Parser implementieren (Format-Spezifikation: https://www.swisschess.ch/media/attached/2021/09/TRF-16.pdf)
- Server Action: `importTRF()` in `src/lib/actions.ts`
- Upload-Component mit Drag & Drop
- Validierung gegen Zod-Schema
- Fehlerhandling für malformed TRF-Dateien

**Aufwand:** 3-4 Tage

### 2. Swiss-System Paarungen (Release 2 - Should)
**Warum wichtig:** Die meisten Vereinsturniere nutzen das Schweizer System.

**Option A: bbpPairings Integration (empfohlen)**
- bbpPairings als Docker-Service hinzufügen
- Node.js-Wrapper für CLI-Aufrufe
- TRF-Export → bbpPairings → TRF-Import Flow

**Option B: JaVaFo Integration**
- Java-basierte Alternative
- Ähnlicher Workflow

**Aufwand:** 4-5 Tage

### 3. SEPA-XML Export (Release 4 - Must)
**Warum kritisch:** Kassenwart benötigt dies für Beitragseinzug.

**Technische Umsetzung:**
- SEPA-XML Generator (pain.008.001.02 Format)
- Mandatsverwaltung erweitern
- Export-Dialog mit Datumsauswahl
- Server Action: `generateSepaXML()`

**Aufwand:** 2-3 Tage

### 4. E-Mail-Versand konfigurieren (Release 1 - Must)
**Warum kritisch:** Passwort-Reset und Einladungen funktionieren nicht ohne SMTP.

**Technische Umsetzung:**
- SMTP-Konfiguration in `.env`
- E-Mail-Templates verifizieren
- `nodemailer` Transport konfigurieren
- Fallback auf Entwicklungsmodus (Konsolenausgabe)

**Aufwand:** 1 Tag

## Technische Hinweise

### Priorität 1: TRF-Format
Das Tournament Report File (TRF) Format ist essentiell für SwissChess-Kompatibilität. Key fields:
- `012` Tournament name
- `032` Date
- `001` Player ID
- `022` Player name
- `013` Federation
- `041` Birthdate
- `051` Gender
- `033` Points
- `041` Rank

### Priorität 2: bbpPairings
Open-Source Pairing Engine: https://github.com/BieremaBoyzProgramming/bbpPairings
- Unterstützt Swiss, Dubov, Lim Systeme
- CLI-basiert, leicht zu integrieren
- TRF als Input/Output

### Priorität 3: SEPA
- pain.008.001.02 für Lastschriften
- Mandatsreferenz muss eindeutig sein
- Erste Sammlung vs wiederkehrende Sammlung
