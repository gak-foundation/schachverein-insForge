# Rollen- und Rechtesystem - Vervollständigung

## Zusammenfassung
Das Rollen- und Rechtesystem ist bereits grundlegend implementiert:
- 8 Rollen im Database-Enum: admin, vorstand, sportwart, jugendwart, kassenwart, trainer, mitglied, eltern
- 23 Permissions in PERMISSIONS constant mit Role-Mapping
- Datenbank-Schema mit `role` und `permissions` JSONB Feldern
- Auth.js Integration mit JWT/Session
- Sidebar mit Permission-Filtering

## Ziel
Vervollständigung der Admin-Verwaltungsoberfläche zur Benutzer- und Rollenverwaltung.

## Aufgaben

### 1. Admin-Benutzerliste erstellen
- Datei: `src/app/dashboard/admin/users/page.tsx`
- Alle Benutzer mit Rolle, Name, Email anzeigen
- Suche und Filter nach Rolle
- Edit-Link zu Benutzer-Detailseite

### 2. Admin-Benutzer-Edit-Seite erstellen
- Datei: `src/app/dashboard/admin/users/[id]/edit/page.tsx`
- Rolle ändern (Dropdown mit 8 Rollen)
- Zusätzliche Berechtigungen verwalten (Checkboxen für alle 23 Permissions)
- Speichern mit Server Action

### 3. Server Actions für Rollen-Management
- Datei: `src/lib/actions.ts` (erweitern)
- `updateUserRole(userId, role, permissions)`
- Audit-Logging bei Änderungen
- Revalidate nach Update

### 4. Sidebar Admin-Menü hinzufügen
- Datei: `src/components/layout/sidebar.tsx`
- Admin-Gruppe mit "Benutzer"-Link
- Nur sichtbar für ADMIN_ROLES Permission

### 5. Route Protection für Admin-Routen
- Datei: `src/proxy.ts` (Middleware)
- Admin-Seiten nur für ADMIN_USERS Permission zugänglich

### 6. Zod-Validierung
- Datei: `src/lib/validations/index.ts`
- Schema für User-Update mit Rollen-Validierung

## Rollen-Übersicht (Bereits definiert)
| Rolle | Berechtigungen |
|-------|---------------|
| admin | Alle |
| vorstand | members.* (r/w), finance.read, events.*, admin.audit |
| sportwart | teams.*, tournaments.*, games.*, dwz.*, events.read |
| jugendwart | members.read:youth, tournaments.read, events.*, parent.dashboard |
| kassenwart | finance.*, sepa, members.read |
| trainer | members.read, teams.read, tournaments.read, games.*, events.read |
| mitglied | members.read (eingeschränkt), teams.read, tournaments.read, games.read |
| eltern | parent.dashboard, events.read |
